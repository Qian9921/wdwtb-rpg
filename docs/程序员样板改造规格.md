# 程序员 Act2 样板改造规格 · "让玩家知道自己喜不喜欢这职业"

> 目标(你的原话):玩完能真正找到职业方向、体验真实的职业环境和生活,**至少让玩家知道自己会不会喜欢这个职业**。
> 本规格只改 **程序员 act2 一幕** 作可玩样板。味道对了再推广。

---

## 一、诊断:现在为什么做不到

游戏是"头尾对、中间空":
- **头(测评)**:7道情境题很好,能测出人。✅
- **尾(报告)**:心之画像 + 职业契合 + 体感信号,设计意图完全对准"喜不喜欢"。✅
- **中间(体验)**:断了。剧情用**旁白替玩家总结感受**("你有点想笑""既兴奋又发怵"),**选择近半无营养**(act2 有13个选择,只7个带tag/axes),做工单是**换皮通用小游戏**。

结果:报告想说"结合你这几天的体感",可**体感没被真正制造出来**,只能退回算法分和模板句。要结算的账户是空的。

## 二、底层机制已全部就绪(不造新轮子)

样板不新增系统,只"喂饱"已有的:
- `ChoiceLog`(记录每次选择的 tag/axes)——反应记录器
- `PersonalityAxes`(4轴:独立↔协作 / 随性↔规划 / 理性↔共情 / 求稳↔冒险)——反应模式
- `InsightCodex`(tag累计→解锁"感悟卡")——啊哈时刻
- `CareerFit.careerBodySignals`(状态→喜欢/适合信号)——喜不喜欢的判断
- `EndingScene`(把上面全部聚合成报告 + AI画像)——回放

## 三、样板做三件事

### 改造1:叙事从"讲述"改成"经历"(去旁白化)

原则:**不告诉玩家他的感受,让玩家自己产生感受。**删掉所有"你有点想笑/既兴奋又发怵/你意识到"这类替玩家总结的旁白,改成呈现具体的情境细节(时间、进度条、对方的表情、群消息),让情绪从玩家心里长出来。

对照(act2 `a2_reqchange`):
- 旧:「产品经理小林 @你:能不能今天就改一下?」**你手上的活儿还没完。这是你第一次撞上「需求变更」。**(← 旁白点破)
- 新:(你正盯着写了一半的登录接口,进度条卡在 60%。)群里「叮」——小林:「登录页老板临时想加个『第三方登录』,今天能上不?就一个按钮的事。」*(右下角:16:40。你今天的活还没交。)*(← 让玩家自己看到冲突)

### 改造2:每个选择都"有营养"+ 有真实后果

act2 全部13个选择都配齐 `tag` + `axes` + 差异化 `effects`,让每次选择都喂记录器。**关键:同一个选择的后果要能被玩家感到**——不只是数值,还有"明天开局状态""NPC关系""老陈是否点头"这类可见回响。

示例(`a2_reqchange` 三选项各不同后果):
| 选项 | tag | axes | effects | 可见回响 |
|---|---|---|---|---|
| 「行,我加班搞」 | `please` `overwork` | empathy+1 | health-8 stress+5 | 留到九点;小林群里@全组夸你;明天精力-开局 |
| 「今天不行,排明天」 | `boundary` | empathy-1 risk-1 | health守住 | 小林沉默几秒「…好吧」;你不确定她高不高兴 |
| 「一个按钮但要接SDK,得两天」 | `principle` `estimate` | plan+1 empathy-1 | passion+2 | 老陈路过,微微点头(专业被看见) |

### 改造3:埋"职业试金石"——测"喜不喜欢程序员"

新增一类**职业共鸣 tag**(区别于现有的人格 tag),专测玩家对"程序员核心活动"的态度。act2 至少埋3个试金石时刻:

1. **造物时刻**(`a2_task_dev` 从零写接口):
   - 「盯着空编辑器,手痒,想马上敲出第一行」→ tag `love_building`(造物的快感)
   - 「一片空白让我发怵,更想有人给个模板照着改」→ tag `prefer_structure`
2. **攻坚时刻**(接口调不通/bug藏得深):
   - 「越查不出越上头,不解决睡不着」→ tag `love_debugging`(死磕的快感)
   - 「查半天没头绪,烦躁,想撂下换点别的」→ tag `flee_hard`
3. **上线时刻**(`a2_start` 你的功能130人用了):
   - 「130个陌生人用我写的东西——这感觉值了」→ tag `love_impact`
   - 「哦,130个。那…下一个需求是啥」→ tag `neutral_impact`

### 新增感悟卡(接 InsightCodex,让结局能回放)

在 `INSIGHTS` 数组加"职业共鸣"类(累计≥2次即解锁):
- `ins_love_code`「和系统死磕会上瘾」:tag `love_building`/`love_debugging` 合计≥2 → "你一次次为'把它跑通'兴奋——写代码这件事本身,可能就是你喜欢的。这是'适合'之外,更难得的'喜欢'信号。"
- `ins_code_neutral`「代码是工具,不是热爱」:tag `prefer_structure`/`flee_hard`/`neutral_impact` 合计≥2 → "你把代码当把趁手的工具,但它没让你上头。也许你更该试试离'人'或'创意'更近的方向——热爱值得再找找。"

### 结局如何用(闭环)

`careerBodySignals` 已有状态信号;再让报告的**职业契合板块**读这两张新感悟卡:
- 解锁 `ins_love_code` → 契合文案倾向"喜欢信号偏强,这行值得你认真考虑"
- 解锁 `ins_code_neutral` → "适合度尚可但热爱没点着,建议再试一条对照线(如设计/产品)"

这样报告回答的就不再是泛泛的"你是XX的人",而是**"你玩这几天,遇到写代码/攻坚/上线时,是来劲还是想逃——所以你(大概)喜欢/没那么喜欢程序员"**。

## 四、验收(样板是否成功)

一个测试者玩完程序员 act2 样板,应该能:
1. 明确说出"我遇到P0/攻坚时是____(来劲/想逃)"——**体感被制造出来了**
2. 结局报告能引用他的真实选择("你3次主动接下最难的bug")——**反应被记录了**
3. 报告直接回答"你(大概)喜欢/没那么喜欢这行,因为____"——**回答了你要的问题**
4. 玩家愿意"再试一条线对照"——**引导到下一步探索**
